任务队列

前端大部分时候都是基于回调的，如果有这么一个需求，比如说，我这有 100 个请求，我要一个一个的发送。

解决方案：

  1. 一个一个调用，直到调完 100 个
  2. 用一个队列，塞进去 100 个，让队列自己去处理 这个 100 个任务

下面是一个简单的任务队列实现：

// 任务队列
// 队列的每一个任务必须是 Promise 的
// @params task 任务函数，返回一个Promise
// @params ctx 任务函数，调用时的上下文，默认是 TaskQueue
// @params arguments 任务函数，调用时的参数，默认是 undefined
// @params success 任务函数，调用成功后的回调
// @params fail 任务函数，调用失败后的回调

// { task: Function, ctx:Any, arguments:Array, success:Function, fail:Function, finally:Function }
export class TaskQueue {
  static queue = [];
  static add(item = {}) {
    if (item.task && item.task instanceof Function) {
      TaskQueue.queue.push(item);
    }
    if (TaskQueue.queue.length === 1) {
      TaskQueue.run();
    }
  }

  static run() {
    if (TaskQueue.queue.length === 0) {
      return;
    }
    let PromiseTask = TaskQueue.queue.shift();
    if (PromiseTask && PromiseTask.task instanceof Function) {
      PromiseTask.task
        .apply(PromiseTask.ctx || this, PromiseTask.arguments)
        .then(res => {
          typeof PromiseTask.success === "function" && PromiseTask.success(res);
        })
        .catch(error => {
          typeof PromiseTask.fail === "function" && PromiseTask.fail(error);
        })
        .finally(() => {
          typeof PromiseTask.finally === "function" && PromiseTask.finally();
          TaskQueue.queue.length > 0 && TaskQueue.run();
        });
    }
  }
}
