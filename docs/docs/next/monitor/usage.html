实时大盘

概述

实时大盘是所有监控项的入口及实时状态监控。

实时大盘页面实时更新所有监控项 最近一小时 或 一天 的关键指标值；可在发版、重要活动等关键时间点实时关注页面监控状况。

功能说明

1、指标实时动态

如图所示，在实时大盘中可以查看指标的实时动态。

  • 默认显示近一小时的指标数据，每分钟实时动态更新
  • 通过右上角切换按钮，可选择查看 近一小时 或 今日 实时数据
  • 点击页面中任意监控项卡片，可跳转到相应的分析页

2、应用设置

点击左上角应用名称的设置按钮可进入到应用设置页，配置应用名称、自定义监控项等。

3、切换监控项

一个应用通常情况下都有很多个监控项，那么我们要如何方便的在各监控项之间切换呢？

如下图所示，例如在 页面性能 分析页面，此时若想切换到 JS 异常 分析页面怎么操作呢，有如下两种方式。

  • 快捷切换：将鼠标移至图中标记 ① 处的「页面性能」，从页面展开的监控项中选择 JS 异常 即可
  • 大盘切换：点击图中标记 ② 处的「实时大盘」回到实时大盘，再选择 JS 异常卡片进入到 JS 异常分析页面

报警监控

概述

报警监控支持开发者对所有监控指标进行报警配置，配置页面性能、异常次数、异常比率、PV 等阈值，当触发报警规则时根据预设的报警方式发送提醒。

功能说明

支持多种时间跨度、阈值规则、过滤条件，配置灵活。

  • 时间跨度支持 1~60 分钟任意时长（环比为 1~30 分钟）
  • 支持多个维度条件（如：页面 URL、网络类型、运营商、国家、省份等）
  • 支持同时满足多条规则或满足其中一条规则的报警
  • 支持大于、小于阈值报警、也自持环比上涨或下跌报警
  • 支持设置 静默期 (即：同一条报警规则，多久内部用重复通知，避免造成骚扰)
  • 支持邮箱、钉钉群等通知提醒方式

1、在监控指标页面，点击 页面左侧导航栏 报警 按钮，进入报警设置页面。

2、在报警设置页面，点击 页面右上角的 创建 按钮，开始配置报警规则。

3、按照业务需要，设置报警规则和通知提醒方式即可。

自定义监控

概述

按照自定义监控协议上报即可实现一个新的监控指标，可用于业务错误、业务关键节点等监控场景。

功能说明

1、添加自定义监控

在 产品设置-监控项管理，点击 "+添加监控项"添加一个监控项，如图所示。

其中：

  • 每个监控项以 category 区分（平台自动生成，不可修改）
  • 支持一个 msg 字段用于在同一监控项中下不同子业务区分
  • 支持一个「率」和一个「均值」字段的统计（后面将会说到）
  • 支持 c1~c5 总共 5 个辅助性字段，可在高级搜索中聚合，因此建议打点可枚举类目数据

2、 代码中增加打点

如果你的项目已经初始化了 SDK，那么直接通过 report() API 上报数据即可，若未初始化 SDK，那可以参考《探针 API 使用入门》先初始化 SDK。

示例代码。

__wpk.report({
  category: 101, //平台创建时，生成的code

  msg: "source1",

  w_succ: 1, // 用于计算率的字段，不需要可不填

  wl_avg1: 100, // 可用户计算均值的内容，不需要可不填

  c1: "xx", // 业务字段，用于在日志中展示和辅助分析，没有可不填。

  c2: "xx"
});

3、设计合理的监控项

自定义监控项很自由，但是设计得合理才更能帮助分析，下面是使用建议。

  • 适合创建独立监控项的场景

独立不相干的信息，如：

  1. 项目中初始化一个 SDK，但可能会失败，适合创建一个「sdk 初始化失败」监控项
  2. 页面资源命中本地缓存的概率，适合创建一个「资源缓存命中率」监控项
  • 适合在监控项中区分不同子业务的场景 相关信息中的不同场景，如：
  1. sdk 初始化失败，有多个 sdk 需要区分，那么 msg 字段的内容设置为不同的 「sdk 名称」 即可
  2. 页面资源缓存命中率，有多个资源要区分，那么 msg 字段的内容设置为不同的 「资源名称」 即可

示例代码。

// a资源

__wpk.report({
  category: 100,

  msg: "a.js",

  w_succ: 1
});

// b资源

__wpk.report({
  category: 100, // 必填，一般是100以上的数字，1~99为预留为系统使用

  msg: "b.css",

  w_succ: 0 // 成功为1，失败为0
});
